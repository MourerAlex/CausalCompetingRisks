## Blocking for Stensrud

### 1. No identification-assumptions section in `separable_effects()` roxygen
- File: `R/separable_effects.R:1–87`
- Current: roxygen describes inputs/outputs but does not state any identifying
  assumption (exchangeability, positivity, consistency, generalized
  decomposition assumption, dismissible component conditions Δ1/Δ2).
- Locked spec (`dev/API_v1_LOCKED.md:12`) says v1 is full isolation only.
- Why blocking: the framework's author would expect to see Δ1/Δ2 named
  on the entry point. Their absence reads as the implementer not
  recognizing they are required.
- Suggested: `@section Identification (full isolation v1)` block listing
  exchangeability / consistency / positivity / generalized decomposition
  / dismissible component conditions, with one-line Stensrud 2020 cite
  per item.

## Would bother an epidemiologist

### 2. "direct"/"indirect" used without "separable" qualifier
- Files: `R/contrasts.R:17–18,25,30,34–46`, `R/plot.R:21–22,134–135,163,509–512`,
  `R/print.R:74–78,97–99`
- UL rule: bare "direct"/"indirect" is loaded mediation vocabulary
  (Pearl/VanderWeele natural/controlled effects). Stensrud's are
  qualitatively different (no cross-world contrasts, competing events
  preserved). Always qualify as **separable** direct / indirect.
- Suggested: prefix with `sep_` consistently in code labels; use
  "separable direct effect (SDE)" / "separable indirect effect (SIE)" in
  prose.

### 3. `"ipw1"` / `"ipw2"` method names vs "Rep 1" / "Rep 2" prose
- Files: `R/separable_effects.R:21,103,223,265`, `R/bootstrap.R:25,36,98`,
  `R/contrasts.R:26`, `R/separable_ipw.R:3,6,8,13,36`
- Mismatch: API tokens are `ipw1`/`ipw2`; roxygen and comments say
  "IPW Rep 1" / "Rep 1" / "Representation 1". Locked API
  (`dev/API_v1_LOCKED.md`) collapses both into single `"ipw"` token,
  with internal Rep 1/Rep 2 returned as `ipw_rep1`/`ipw_rep2`.
- Suggested: collapse user-facing tokens to `"ipw"`; output and roxygen
  use "IPW Rep 1" / "IPW Rep 2" consistently for the two representations.

### 5. `y_flag`/`d_flag`/`c_flag` columns not glossed
- Files: `R/data_prep.R:14,149–152`, `R/validate.R:288,313,337`,
  `R/separable_effects.R:14`
- Column names work but roxygen never ties them to Y (primary event),
  D (competing event), C (censoring). Newcomer can't tell what `y` is.
- Suggested: roxygen `@details` on `to_person_time()`:

  C → D → Y enforced.
